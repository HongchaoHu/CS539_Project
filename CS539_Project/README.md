# Data Analysis Agent

Data Analysis Agent is a FastAPI application that uses Google Gemini to generate
Python analysis code, execute that code against uploaded CSV data, and return
text summaries plus saved visualization artifacts. The same backend also exposes
an API-first machine-learning route that asks Gemini for self-contained ML code
and executes it through the same visualization runtime.

## What The Project Does

- Accepts a CSV upload and a natural-language analysis question.
- Collects lightweight dataset metadata and sends that context to Gemini.
- Executes generated Python code in a controlled runtime with pandas,
	matplotlib, seaborn, numpy, and optional sklearn helpers.
- Saves generated figures under `outputs/` and serves them through the API.
- Exposes a backend ML endpoint for algorithm demonstrations and generated
	machine-learning visualizations.

## Prerequisites

- Python 3.10+
- A Google Gemini API key

## Installation

1. Create and activate a virtual environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Create a `.env` file in the project root:

```env
GEMINI_API_KEY=your_api_key_here
GEMINI_MODEL=gemini-1.5-flash
TEMPERATURE=0.7
MAX_TOKENS=2048
```

`GOOGLE_API_KEY` is also accepted if that is already how your environment is configured.

## Running The Application

Start the server from the project root:

```bash
uvicorn app:app --reload
```

Useful URLs after startup:

- `http://localhost:8000/` for the web UI
- `http://localhost:8000/docs` for Swagger UI
- `http://localhost:8000/health` for a quick health check

## API Overview

- `POST /upload-analyze`
	Upload a CSV and run natural-language exploratory analysis.
- `POST /analyze`
	Run analysis against a CSV file path already available on disk.
- `POST /ml-solution`
	Ask Gemini for self-contained machine-learning code and execute it.
- `GET /visualization/{filename}`
	Retrieve saved chart images.
- `GET /artifact/{analysis_id}`
	Download a saved JSON artifact.

## Frontend And ML Flow

The current browser UI is wired to the CSV upload flow. The ML route is already
available on the backend through `POST /ml-solution`, but it is not yet exposed
as a separate browser mode in `static/index.html` and `static/js/app.js`.

## Maintenance Guidance

- Keep `app.py` thin. New business logic should live in `src/agent.py` or a tool module.
- Keep execution concerns in `src/tools/visualization.py`. That file should stay focused on runtime context and artifact capture.
- When changing Gemini prompts, update both the prompt text and this README so the documented runtime contract stays accurate.
- Prefer adding a small, well-placed docstring or boundary comment over repeating obvious inline comments.
- If you add more generated-code capabilities, document the execution context explicitly so future maintainers know which objects are injected by the runtime versus generated by Gemini.
